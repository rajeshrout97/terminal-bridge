Features

Why Terminal Bridge?

Built for developers who want their AI agent to control a remote Mac without SSH tunnels, port forwarding, or config files.

🤖

Model Agnostic

Works with Cursor, Claude CLI, Ollama, LM Studio, aider, or any tool that speaks HTTP, MCP, or stdio. No vendor lock-in.

🔒

Secure by Default

HMAC-SHA256 challenge-response auth, TLS encryption on all connections, and keys stored in macOS Keychain — never in plaintext.

⚡

Zero Config for Cursor

The setup wizard auto-configures Cursor's MCP. Restart Cursor and your agent can immediately control the remote Mac.

🌐

LAN + Internet

Works over WiFi out of the box. Add Tailscale for free global access, or use the built-in relay server.

🔄

Background Service

The remote agent runs as a launchd service — auto-starts on login, survives reboots, zero daily maintenance.

📡

Six Interfaces

CLI, REST API, Python SDK, MCP server, stdio pipe, and interactive terminal. Use whatever fits your workflow.

Interfaces

Use it your way

Six ways to talk to your remote Mac. Pick the one that fits.

Interface Best For Example
CLI Shell scripts, quick commands tbridge exec "ls -la"
REST API Ollama, LM Studio, HTTP clients POST localhost:9876/api/exec
Python SDK Custom automation scripts from terminal_bridge import RemoteTerminal
MCP Server Cursor IDE agent Auto-configured
Stdio Pipe Claude CLI, aider echo '{"tool":"exec",...}' | tbridge pipe
Virtual Terminal Interactive SSH-like access tbridge connect --terminal
Setup

Three commands to get started

Install on both Macs, run the setup wizards, done.

01

Install

Install Terminal Bridge on both Macs using pip or pipx.

pip install terminal-bridge
02

Setup remote

On the Mac you want to control, run the setup wizard. It generates a pairing code.

tbridge setup remote
03

Pair locally

On the Mac where your AI runs, paste the code. Cursor MCP is auto-configured.

tbridge setup local <CODE>
